#### （2）技術創新與差異化優勢  


1. **感測器可靠度與 BHI/RUL：從「單點告警」升級為「可預測的健康管理」**  
   - 以 Kalman 濾波＋時序深度學習（如 LSTM/TFT）建立 BHI（Battery Health Index）與 RUL（Remaining Useful Life），不僅回報「壞了沒」，更能提前預測「何時會壞」。  
   - 與多數僅看電壓門檻的方案相比，可同時考量溫度、使用型態與傳輸頻率，降低誤判並延長感測器壽命。

2. **NLQ＋語意層：從「看報表」變成「問問題就給答案」**  
   - 設計物流專用語意層（Shipment、Lane、Container、Event 等），並以 NL-to-SQL 將自然語言查詢安全地轉成結構化查詢。  
   - 匯總 BHI、違規事件、路線異常與 SLA 指標，高階主管可直接詢問「本季歐洲線溫濕度違規趨勢？」而非人工拉報表。

3. **環境預測與路線異常偵測：將天候／交通納入解釋，降低誤報**  
   - 針對溫濕度敏感貨品，以多變量時序模型預測未來艙內條件，並將天候與季節做為外生特徵，提升違規提前預警能力。  
   - 路線異常偵測則結合 GPS、停留模式與天候／交通資訊，區分「合理延誤」與「異常繞路／竊盜風險」，優先處理真正高風險案件。

# 貳、計畫內容與實施方法

【摘要】本節由產業痛點引導 AI 解法與工作包，明確串連「痛點→解法→里程碑→KPI」，並補強資料治理與資安合規，確保量測可行與可追蹤。
## 一、計畫緣起與產業痛點

本計畫針對智慧物流場域，優先處理影響可靠度、效率與客戶體驗的核心痛點，並以分項計畫（A–E）承接。
- 痛點一｜感測器可靠度不足（電池衰退、環境干擾、資料遺失）→ 需可預測的電池健康與壽命管理（分項計畫A）。
- 痛點二｜報告產出耗時且缺乏互動（人工彙整、決策延遲）→ 需按需報告與自然語言查詢（分項計畫B）。
- 痛點三｜環境敏感貨品的違規風險（如咖啡豆濕度）→ 需環境條件預測與違規風險預警（分項計畫C；外部天候與季節變化會影響貨艙內溫濕度動態，故將天候作為環境預測模型之外生特徵，以提升違規提前預警能力）。
- 痛點四｜路線繞行與潛在竊盜（GPS偏離、天候與交通影響）→ 需路線異常與竊盜偵測（分項計畫D；天候與交通會影響繞路與停留模式，納入相關資訊可區分合理延誤與真正異常，降低誤報並強化竊盜偵測能力）。

### （2）分項計畫

#### 分項計畫A｜智慧感測器可靠度優化

目標在於將感測器可靠度由約 90% 提升並降低非計畫性停擺，聚焦建置可用的電池健康指標（BHI）與剩餘壽命（RUL）。

**A1. 目標與成功標準（Objectives & Success Criteria）**<br>
   - 可靠度（Availability）≥95%、RUL MAE ≤ 10 天、電池壽命延長 ≥ 40%（資料完整性 ≥ 95%）。
**A2. 取得與準備資料（Data Acquisition & Preparation）**<br>
   - 使用裝置電壓、溫度、傳輸頻率與維護紀錄等資料，補齊缺漏並建立健康趨勢。
**A3. 建立數據集（Dataset for Modeling）**<br>
   - 整合歷史資料與事件標註（故障/維護），形成可回溯的訓練與驗證集。
**A4. 建模與訓練（Modeling & Training）**<br>
   - 採用適合時序預測的方法，並以簡單規則輔助，確保穩定與可落地。
**A5. 評估與優化（Evaluation & Optimization）**<br>
   - 以準確度與預測誤差為主進行評估，提供簡明可解釋的結果說明。
**A6. 佈署與展示（Deployment & Showcase）**<br>
   - 提供 BHI/RUL API 與儀表板，支援預換電與維保排程，並與自適應採樣策略介接。

#### 分項計畫B｜按需報告與自然語言查詢（NLQ）

讓高層與營運人員以自然語言即時取得洞見，並自動生成週/月報與優先警示彙報。

**B1. 目標與成功標準**<br>
   - 報告製作時間下降 ≥ 50%；NLQ P95 延遲 < 45 秒；正確率 ≥ 90%。
**B2. 取得與準備資料**<br>
   - 建立指標語義層與資料字典，串接時序/事件資料與外部摘要。
**B3. 建立數據集**<br>
   - 彙整常見查詢樣式與報表模板，建立可維護的資料目錄。
**B4. 建模與訓練**<br>
   - 以白名單與唯讀權限連接資料庫，限制查詢範圍並避免對數值事實做臆測。
   - 建立 Executive Report Agent（彙報智能體）：編排指標擷取→語意摘要→異常說明→建議行動，支援多輪追問（Follow-up Query）與上下文維持。
**B5. 評估與優化**<br>
   - 以正確率與延遲為主監測，建立拒答與審計機制，持續調整模板。
**B6. 佈署與展示**<br>
   - 上線 NLQ API、Interactive Dashboard、Executive Auto-Report Agent 與定期報表；提供高階彙報＋互動問答（Follow-up / Drill-down）。

#### 分項計畫C｜環境條件監測與預測

針對溫濕度敏感貨品（如咖啡豆），建置條件預測與違規風險預警機制，降低變質與索賠。

**C1. 目標與成功標準**<br>
   - 達成合理預測誤差；違規提前預警率 ≥ 70%；違規率下降 ≥ 30%。
**C2. 取得與準備資料**<br>
   - 使用溫濕度與位置等資料，搭配路線、季節與外部天候等背景資訊（天候作為外生特徵）。
**C3. 建立數據集**<br>
   - 建立多變量資料集與標註，覆蓋主要路線與艙體情境。
**C4. 建模與訓練**<br>
   - 採用穩健的時序預測方法，納入簡單先驗與邏輯檢核，並將外部天候做為背景變數。
**C5. 評估與優化**<br>
   - 以預測誤差與違規檢出為主評估，持續監測漂移。
**C6. 佈署與展示**<br>
   - 提供預警 API 與處置建議，並整合至客戶報告（對應分項計畫B）。

#### 分項計畫D｜竊盜偵測與路線監控

透過路線偏離、可疑停留與外部路況/天候建模，辨識潛在繞路與竊盜，並優化警示優先序。

**D1. 目標與成功標準**<br>
   - 偵測 AUC ≥ 0.90、F1 ≥ 0.85；ETA MAE 改善 ≥ 25%；警示誤報率 ≤ 15%（含因天候/交通之合理延誤過濾）；關鍵警示回應時間 < 30 分鐘。
**D2. 取得與準備資料**<br>
   - 使用 GPS 位置、速度、停靠點與外部路況/天候摘要。
**D3. 建立數據集**<br>
   - 以行程切片與異常標註建立資料集，補充路網特徵。
**D4. 建模與訓練**<br>
   - 以時序與地理行為分析識別偏離、可疑停留與未授權開箱，並結合交通/天候上下文以區分可解釋延誤與真正異常，設定警示優先序。
**D5. 評估與優化**<br>
   - 以 AUC/F1 與延遲監測成效，進行 A/B 測試。
**D6. 佈署與展示**<br>
   - 異常/竊盜警示服務；路線回放、根因分析；MCP Agent 編排整合（跨分項）。

#### 分項計畫E｜跨分項整合、技術驗證與國際輸出

將分項計畫 A–D 之技術與指標整合為穩定、可重現之示範系統，完成跨模組技術驗證與 1–2 場國際展示，形成出口型白皮書與商務轉換流程。

**E1. 目標與成功標準（Objectives & Success Criteria）**
   - 端到端跨模組整合（A–D 之 API/事件/指標）：資料同步成功率 ≥ 98%，跨模組告警對齊 P95 延遲 < 5 秒。
   - 國際展示 1–2 場（如 TOC Asia／LogiMAT／Transport Logistic），取得 ≥ 2 份 LOI 或 POC 協議。
   - 產出技術白皮書 1 份、技術解說影片 1 支、專利亮點包 1 份（含可解釋性案例與方法章節）。
   - 專家／媒體評分 ≥ 4/5；Demo 整合缺陷率呈連續 3 週下降趨勢。

**E2. 整合與驗證項目（Integration & Validation Items）**
   - 指標層整併：BHI／RUL、誤報率、違規預警率、路線異常 AUC、ETA MAE 等統一語義層（semantic layer）。
   - 事件流與資料一致性：時序對齊（time alignment）、時區處理、資料譜系（data lineage）與校驗器（validators）。
   - 效能與穩定性：NLQ 端到端查詢延遲 P95 基準值 < 60 秒，優化目標 P95 ≤ 10 秒，最終目標 P95 ≤ 2 秒；警示處理管線 P95 < 5 秒；可用性 ≥ 99.5%。
   - 安全與合規：GDPR／DPIA 檢核；區域化部署／聯邦學習（FL）；審計日誌（audit log）。

**E3. 主要工作項目（Work Packages, WPs）**
   - E3-1 Demo／PoC 統整：前端 Dashboard、API Gateways、MCP Agents、資料目錄與數據字典。
   - E3-2 可解釋性套件：針對每類警示輸出解釋報告（特徵貢獻、決策路徑、可信度區間）。
   - E3-3 白皮書與標準化：方法、指標、驗證協議與復現步驟；對齊 EU AI Act 與 O-RAN/AI-RAN 延伸參與（路線圖與介面層，非本期交付）。
   - E3-4 國際展示作業：展前演練、現場腳本、備援拓撲（線上真實流／離線回放）、收單流程與法務模板（NDA/LOI/POC）。
   - E3-5 商務轉換：線索管理（CRM）、定價與方案包、合作模式（Arviem 通路／直銷／授權）與投後追蹤。

**E4. KPI 與查核（Milestones & KPIs）**
   - 技術整合：資料同步成功率 ≥ 98%；跨模組告警延遲 P95 < 5 秒；NLQ 正確率 ≥ 90%，延遲 P95 基準值 < 60 秒，期末目標 < 45 秒（後續最佳化階段可再評估 ≤30 秒、進階 ≤10 秒、長期 ≤2 秒 之分級）。
   - 展示與輸出：國際展示 ≥ 1 場；LOI／POC ≥ 2；白皮書／影片／網站頁面上線；展後 30 日內有效線索 ≥ 10。
   - 品質與穩定：展前兩週零 P1 缺陷；整合缺陷率按週下降；服務可用性 ≥ 99.5%。

**E5. 風險與因應（Risks & Mitigations）**
   - 展會檔期或夥伴延遲 → 備選展會或線上發表；與合作方場域聯名發佈；內容改以白皮書預覽版先行。
   - 現場網路不穩 → 雙備援：熱身資料集 + 線上接真實流；離線緩存與回放；降級方案（只讀模式）。
   - 合規審查未完成 → 使用匿名／合成資料、區域化部署；審查完成後切換生資料；第三方審查排程前移。
   - Demo 漏洞與穩定性 → 演練清單、Rollback Playbook、灰階流量與只讀帳號。

**E6. 佈署與展示交付（Deliverables）**
   - 上線環境：雲端 MVS（最小可用服務）＋監控告警儀表板＋災備拓撲。
   - 展示資產：技術白皮書、技術影片、案例手冊、媒體素材包（含圖表與指標說明）。
   - 法務／商務：LOI／POC 模板、NDA、定價與授權條款摘要、合作模式說明。
   - 驗收文件：查核點報告、第三方驗證摘要、展後追蹤成效報告（線索與轉換率）。

## (三)總收入結構分析：硬體 vs. 服務佔比（2026-2029）


| 項目           | 2026 (USD) | 2027 (USD) | 2028 (USD) | 2029 (USD) |
| ---------------- | ------------ | ------------ | ------------ | ------------ |
| 總收入         | $93,000    | $812,000   | $1,342,000 | $1,760,000 |

| **服務佔比**   | **19.35%** | **20.21%** | **21.19%** | **23.08%** |

### 核心營運說明與分析：

1. **硬體主導營收：** 在整個預測期內 (2026-2029)，IoT 裝置銷售收入 (硬體) 都是公司收入的主要來源，每年佔比接近八成。這表明公司在初期階段的商業模式高度依賴於硬體產品的一次性銷售。ItracXing 的營收預測與風險將與 Arviem 的通路執行力（合作進度延遲）及市場對$1000/套裝置的接受度（競爭對手價格戰）緊密相關。
2. **服務收入重要性提升：** 儘管硬體佔比高，但服務相關收入 (b+c) 的佔比呈穩定微幅上升趨勢 (從 19.35%增長至 23.08%)。這與應對措施中提到的**「建立多元化收入來源（顧問服務、技術授權）」**相呼應。服務收入（b） 代表 AI-MaaS 或 LaaS（租賃即服務）的經常性收入，雖然在總收入中佔比較小，但由於其高毛利特性（硬體毛利率 50%，服務毛利通常更高），其對公司利潤和估值的貢獻可能比其營收佔比更為重要。
3. **成長驅動因子：**

   - **2026-2027 年：** 總收入呈現爆發性增長 (從$93K 到 $752K)，主要由 IoT 裝置銷售量的大幅增加 (75 套到 600 套) 所驅動。
   - **2027 年後：** 總收入保持穩健增長，同時服務佔比逐漸提高，顯示公司開始從純硬體銷售轉向**「硬體+服務」**的綜合解決方案模式。
